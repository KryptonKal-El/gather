# Toolkit Gap Report — TSK-20260505-01

## TSK-20260505-01 — 2026-05-05

### Context

Ad-hoc story for Gather: installed Vitest and React Testing Library as the first unit-test infrastructure for a React/Vite/Supabase project. Implementation succeeded, `npm run test:run` passes with one smoke test, changed-file lint was clean, tester verification passed, and critic returned zero findings with a caveat that router-level specialist dispatch was not actually available in this API session.

### Findings

#### Existing toolkit issues

- **Priority:** critical
- **Severity:** blocking
- **Scope:** global
- **Remediation:** Concepture-Toolkit
- **File path:** `agents/Concepture-Developer.md`, `agents/Concepture-Tester.md`, `agents/Concepture-Quality.md`, `agents/Concepture-Critic.md`, `agents/Concepture-Builder.md`, `TOOLKIT-CONVENTIONS.md`
- **Issue:** Router agents appear unable to dispatch to project-level specialists in the current API/tool environment, despite the routing contract requiring Builder to route through global routers and routers to dispatch to specialists.
- **Rationale:** `Concepture-Critic` explicitly reported that this session did not expose a sub-agent dispatch tool, so it could not independently execute named specialists and did not perform router-level code review. This directly conflicts with the documented four-router model and can produce false confidence, especially when a critic returns zero findings without actually invoking the specialist.
- **Suggested fix:** Concepture-Toolkit should run a focused investigation against current opencode capabilities. If routers can be given a task/sub-agent dispatch tool, update router frontmatter and instructions accordingly. If nested sub-agent dispatch is impossible, choose and document a new architecture: either flatten Builder so it calls specialists directly, or explicitly redefine routers as persona-adopting workers rather than dispatchers. Then update `TOOLKIT-CONVENTIONS.md`, router agent files, Builder instructions, and project setup guidance so they agree.
- **Act now or defer:** Act now. This is foundational to Builder quality gates and review trust.

- **Priority:** medium
- **Severity:** medium
- **Scope:** global
- **Remediation:** Concepture-Toolkit
- **File path:** `skills/project-bootstrap/SKILL.md`, `docs/project.json` schema/setup guidance
- **Issue:** Project bootstrap allowed `commands.test = "npm test"` to be declared before a test runner existed.
- **Rationale:** The first implementation story had to add missing test infrastructure before the declared command was truthful. A project can look test-ready to Builder even when `npm test` cannot run.
- **Suggested fix:** Update project bootstrap to validate declared commands against installed scripts/dependencies. For JavaScript projects, check `package.json` scripts and test-runner dependencies before writing `commands.test`; if missing, either omit the command, mark it unavailable, or emit a setup warning/story recommendation.
- **Act now or defer:** Defer for the current story because test infra is now installed, but address soon in toolkit setup hardening.

- **Priority:** high
- **Severity:** high
- **Scope:** global
- **Remediation:** Concepture-Toolkit
- **File path:** `skills/session-setup/SKILL.md`, `schemas/session-locks.schema.json` if present
- **Issue:** `skills/session-setup` expects `docs/session-locks.json` to contain `{ "sessions": [] }`, while the project has an existing `{ "$schema": "...", "locks": [] }` shape.
- **Rationale:** A setup skill and existing project state disagree about the lock schema. Without an explicit migration/backcompat rule, sessions may be skipped, duplicated, or written into the wrong top-level collection.
- **Suggested fix:** Decide the canonical lock schema in the toolkit, update `session-setup` and any schema files to match, and add migration/backward-compatibility instructions for projects with the legacy `locks` shape. After that, run a project-level migration for Gather if needed.
- **Act now or defer:** Act now at the toolkit level before more sessions accumulate; defer Gather migration until the canonical schema is chosen.
