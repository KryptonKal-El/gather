# TSK-20260602-193703: Fix — collaborator pencil button does nothing in list detail view

## Request
When a collaborator taps the pencil icon in the top-right of a list view, nothing happens. It should open the edit view.

## Investigation Findings

### Recommended Approach
Move `.sheet(isPresented: $showEditSheet)` from inside `listOptionsMenu` (a computed property that only renders for owners) to the main view body so it is reachable from both the owner's menu and the collaborator's pencil button.

### Confidence
HIGH

### Evidence
- [CONFIRMED] `showEditSheet` is set to `true` in the collaborator pencil button action (line 150)
- [CONFIRMED] `.sheet(isPresented: $showEditSheet)` was chained to `listOptionsMenu` which only renders when `isOwned == true` (line 1301)
- [CONFIRMED] Moving the sheet to the main view body fixes the scoping issue

### Scope
Small (1 file, 3 lines moved)
