# TSK-20260603-154131: Fix /app route breakage caused by postbuild rename + Workbox precache mismatch

## Request
We were looking into why https://gatherlists.com/app doesn't load and gets an error.

## Investigation Findings

### Recommended Approach
- Remove the `postbuild` `mv dist/index.html dist/app.html` script from `package.json`
- Fix the Workbox config in `vite.config.js`: update `globPatterns` to use `index.html` (already correct) but align `navigateFallback` to `/index.html` so the SW can actually serve it from the precache
- Add `index.html` to the `navigateFallbackDenylist`-exempt list so the SW correctly handles navigation to `/app` using the precached `index.html`
- Update `vercel.json` so `/app` and `/app/(.*)` rewrite to `/index.html` instead of `/app.html` (since we're no longer renaming the file)

### Confidence
HIGH

### Evidence
- [CONFIRMED] `package.json` has `"postbuild": "mv dist/index.html dist/app.html"` — renames the file after Workbox has already generated a precache manifest referencing `index.html` (`direct read: package.json`)
- [CONFIRMED] `vite.config.js` workbox config: `globPatterns` includes `index.html`, `navigateFallback: '/app'` — the SW's navigation fallback points to `/app`, which is NOT in the precache, so the SW cannot serve it (`direct read: vite.config.js`)
- [CONFIRMED] `vercel.json` rewrites `/app` → `/app.html` and `/app/(.*)` → `/app.html` — but after the fix, these should point to `/index.html` (`direct read: vercel.json`)
- [CONFIRMED] Workbox aborts SW installation if any precache fetch fails — fetching `/index.html` returns 404 because the file was renamed to `app.html` before deployment (`code trace: workbox precache install behavior`)
- [CONFIRMED] For users with a previously installed SW, the broken `NavigationRoute` intercepts `/app` and fails with "no response" because `navigateFallback: '/app'` isn't cached (`code trace: workbox NavigationRoute + createHandlerBoundToURL`)

### Eliminated Approaches
- **Keep the rename + update Workbox to precache `app.html`**: VitePWA/Rollup does not easily support outputting the HTML entry as a custom filename before Workbox scans it. The rename happens in `postbuild`, after the precache manifest is already baked into `sw.js`. This approach requires either a custom Vite plugin or Rollup input aliasing — more complexity than needed.
- **Keep `navigateFallback: '/app'` and add `/app` to precache manually**: Not straightforward with VitePWA; `globPatterns` can't reliably inject a Vercel-served URL into the precache.
