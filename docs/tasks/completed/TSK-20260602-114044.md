# TSK-20260602-114044: Fix Manage Stores visual — background color and Add button style

## Request
When going into a list and clicking the three-dot menu and selecting "Manage Stores", the add button is hard to see. Two issues:
1. The Stores background color should match the "Edit" view
2. The Add button should use the same green text color

## Investigation Findings

### Recommended Approach
- Add `background: var(--bg-card)` to the `.mobileLayout` in `StoreManager.module.css` so the Manage Stores sheet uses the same card-white background as the Edit modal
- Change the `.circleBtn` style in `StoreManager.module.css` from solid green circle (white text) to green text on transparent background — matching the `.newBtn` pattern used throughout the app (`color: var(--primary); background: none`)

### Confidence
HIGH

### Evidence
- [CONFIRMED] `App.jsx` wraps `StoreManager` in a portal with `.storesModal` / `.storesModalBody` CSS module classes — these classes are missing from `App.module.css`, so the container has no styles (direct code read)
- [CONFIRMED] `StoreManager.module.css` `.mobileLayout` and `.scrollArea` have no background set, so `var(--bg-page)` (#FAFAF7 light / #121212 dark) bleeds through instead of `var(--bg-card)` (#ffffff light / #1e1e1e dark) used by the Edit modal (direct code read)
- [CONFIRMED] `.circleBtn` in `StoreManager.module.css` uses `background: var(--primary); color: #fff` — solid green circle with white text (direct code read)
- [CONFIRMED] Edit modal (`ListSelector.module.css` `.modalCard`) uses `background: var(--bg-card)` (direct code read)
- [CONFIRMED] Other header action buttons (`.newBtn` in `ListSelector.module.css`, `StoreManager.module.css`) use `color: var(--primary); background: none` — green text on transparent (direct code read)
- [CONFIRMED] CSS token `--primary: #3D7A63` (light) / `#5BA08A` (dark) defined in `src/index.css` (direct code read)

### Eliminated Approaches
- Adding the missing `.storesModal` styles to `App.module.css` — unnecessary indirection; applying `background: var(--bg-card)` directly to `.mobileLayout` in `StoreManager.module.css` is simpler and keeps styles co-located with the component
