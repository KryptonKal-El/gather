# TSK-20260603-160005: Fix Apple Sign-In redirect landing on marketing page instead of /app

## Request
After authenticating with "Sign in with Apple", the user gets redirected to the homepage (marketing page) instead of /app.

## Investigation Findings

### Recommended Approach
- Pass an explicit `redirectTo: window.location.origin + '/app'` in the `signInWithOAuth` call in `AuthContext.jsx`
- Manually add `https://gatherlists.com/app` to the Supabase dashboard allowed redirect URLs (Authentication → URL Configuration)

### Confidence
HIGH

### Evidence
- [CONFIRMED] `signInWithApple` calls `supabase.auth.signInWithOAuth({ provider: 'apple' })` with no `redirectTo` option — Supabase falls back to the dashboard Site URL (`https://gatherlists.com/`) (direct read: src/context/AuthContext.jsx)
- [CONFIRMED] Vercel routes `/` → `index-marketing.html` — the marketing page has no React app or `onAuthStateChange` listener, so the `#access_token` fragment in the callback URL is silently dropped (direct read: vercel.json)
- [CONFIRMED] Today's routing fix made this visible — previously `/` could have served `index.html` (the React app) which would have inadvertently caught the token (git log)
- [INFERRED] Supabase dashboard Site URL is `https://gatherlists.com/` not `https://gatherlists.com/app` — needs manual update in dashboard

### Eliminated Approaches
- **Update Supabase Site URL only (no code change)**: Would fix flows with no `redirectTo`, but explicit `redirectTo` in code is more robust and doesn't depend on dashboard config being correct.
- **Add `/auth/callback` route**: Unnecessary complexity — Supabase PKCE flow works fine landing directly on `/app` with the token in the URL fragment.

### Scope
Small (1 file, ~3 lines of code) + 1 manual Supabase dashboard step
