# TSK-20260602-01: Create project conventions documentation

## Summary

Document the coding conventions that exist implicitly in the Gather codebase by creating `docs/CONVENTIONS.md` (index) and six convention layer files under `docs/conventions/`. These files make patterns discoverable by AI agents on future PRDs and reduce inconsistency from convention drift.

## Motivation

The toolkit health check after `prd-list-scoped-stores` flagged the absence of a conventions infrastructure as a high-priority gap. Patterns for React components, testing, database services, state management, Swift/iOS, and dnd-kit all exist in code but are nowhere written down.

## Scope

Create (new files only — no source code changes):

- `docs/CONVENTIONS.md` — index file linking all layer files
- `docs/conventions/react-components.md`
- `docs/conventions/react-testing.md`
- `docs/conventions/database-service.md`
- `docs/conventions/state-management.md`
- `docs/conventions/swift-ios.md`
- `docs/conventions/dnd-kit.md`

## Out of Scope

- No changes to source code
- No new tests (documentation only)
- No changes to `docs/project.json` or `AGENTS.md`

## Acceptance Criteria

1. `docs/CONVENTIONS.md` exists and links all six layer files.
2. Each layer file captures the conventions observed in the codebase during exploration (see session summary).
3. All files are Markdown, concise, and agent-readable (no prose fluff).
