# TSK-20260526-01: Replace category color dot with full-height left bar in iOS native shopping list

## Request
iOS still has the old circle dot UI for category group headers in the shopping list. Replace it with the same full-height 3px left vertical bar treatment that was applied to React web.

## Investigation Findings

### Recommended Approach
- Remove the `Circle()` dot from `categoryHeader(label:color:itemCount:)` in `ListDetailView.swift`
- Wrap the existing `HStack` in an outer `HStack` that places a `Rectangle()` of `width: 3, maxHeight: .infinity` on the leading edge, colored with `Color(hex: color)`
- Remove the `.padding(.leading, 20)` from the inner `HStack` (the bar provides the leading visual anchor; adjust padding to maintain 12px gap between bar and label)
- The level-1 store header dot (`Circle()` at line 506–508, 10×10 pt) is **not** changed — it is the top-level store card, which was intentionally left unchanged in the React PRD

### Confidence
HIGH

### Evidence
- [CONFIRMED] `categoryHeader(label:color:itemCount:)` at `ListDetailView.swift:605–622` renders the category group header with a `Circle().fill(Color(hex: color)).frame(width: 8, height: 8)` dot
- [CONFIRMED] Color is passed as a hex `String` from `pipelineGroupsView` at line 451: `group.color ?? "#9e9e9e"`
- [CONFIRMED] `Color(hex:)` extension exists at `Views/Components/ColorExtension.swift`
- [CONFIRMED] Existing `Rectangle().frame(width: 1)` separator pattern at `ListDetailView.swift:732–735` confirms the idiom is already in use
- [CONFIRMED] No existing left-bar pattern — this is new but consistent with existing Rectangle usage
- [CONFIRMED] Level-1 store header dot at line 506–508 is intentionally out of scope (matches React PRD scope)

### Eliminated Approaches
- `overlay` / `background` modifier — more complex than needed; outer HStack is simpler and idiomatic
- `ZStack` — unnecessary layering for a side-by-side layout
